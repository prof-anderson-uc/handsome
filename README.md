# Handsome the Police Dog

This repository contains the starter files for the first coding exercise in Web Design 1.

In this exercise, you will turn plain text about **Handsome**, Cincinnati's first unofficial police dog, into a basic HTML web page.

This is your first coding exercise, so the goal is not to make the page look visually designed yet. Instead, you are learning how to create the basic structure of a web page using HTML.

---

## Use Canvas for the Full Assignment Instructions

Canvas has the full assignment description, screencast videos, due date, and submission instructions.

This GitHub repository is only where the starter files are stored.

For this exercise, you do **not** need to fork this repository.

You do **not** need to clone this repository.

You do **not** need to commit or push anything.

For now, you are only downloading the starter files as a ZIP file.

---

## What You Will Practice

By completing this exercise, you will practice how to:

- Download starter files from GitHub
- Unzip a project folder
- Open a folder in Visual Studio Code
- Rename a `.txt` file as an `.html` file
- Create the basic structure of an HTML document
- Use headings and paragraphs
- Add images to a web page
- Use `src` and `alt` attributes
- Preview an HTML file in a browser
- Zip and submit a complete project folder in Canvas

---

## Starter Files

This repository includes:

- `handsome.txt`
- `handsome.jpg`
- `handsome-today.jpg`
- `README.md`

The file `handsome.txt` contains the plain text for the article.

During the exercise, you will rename that file as:

`handsome.html`

The image files will be used when you add images to the page.

---

## How to Download the Starter Files

1. Click the green **Code** button near the top of this GitHub page.
2. Choose **Download ZIP**.
3. Save the ZIP file to your computer.
4. Unzip the file before you begin working.
5. Open the unzipped folder in Visual Studio Code.

Do **not** work directly inside the ZIP file. You need to unzip it first.

---

## Important Note for Windows Users

Windows often hides file extensions by default.

That can make it hard to tell whether a file is really named:

`handsome.html`

or accidentally named something like:

`handsome.html.txt`

If you are using Windows, turn on file extensions before starting this exercise. Canvas includes a support page showing how to do this.

---

## What Your Finished Folder Should Include

When you are finished, your project folder should include:

- `handsome.html`
- `handsome.jpg`
- `handsome-today.jpg`
- `README.md`

It is okay if your folder also includes other system files your computer created, but the important files listed above should be there.

Your final HTML file should be named:

`handsome.html`

Not:

- `handsome.txt`
- `handsome.html.txt`

We will use the file name `index.html` later in the course when we start creating homepages. For this first exercise, use `handsome.html`.

---

## Before You Submit

Before submitting your work in Canvas, check the following:

- The file is named `handsome.html`
- The file is not accidentally named `handsome.html.txt`
- The HTML file and both image files are in the same folder
- Your page has the basic HTML document structure
- The main title is marked up as an `h1`
- The section headings are marked up as `h2` elements
- Paragraph text is marked up with `p` elements
- Both images appear when you preview the page in a browser
- Both images include `alt` text
- You zipped the entire project folder, not just the HTML file

---

## Submitting Your Work

When you are finished, compress the entire project folder into a ZIP file and submit that ZIP file in Canvas.

Do not submit only the HTML file. The image files need to be included too, or the images will not work when the project is opened on another computer.

## Reminder

This first page will look plain. That is normal.

HTML gives the page structure and meaning. CSS, which we will learn later, controls visual design such as colors, fonts, layout, and spacing.
